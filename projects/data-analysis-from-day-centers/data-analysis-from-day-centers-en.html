Introduction

This project was developed as part of a freelance job request for data analysis and transformation.

As it was a private job, the developed spreadsheet is the property of the requesting client, so it will not be shown in the results section, but I will include brief excerpts.

The client did not request a technical data analysis report, but rather the creation of various types of charts and tables, so that he could build the report according to his needs.

Below, I will detail the entire process developed, tools used, charts, formulas, etc.

Jorge (fictitious name) requested the following:

"The day centers are institutional modalities aimed at different age groups. I need to analyze only the units that serve elderly people between the years 2019 and 2023. Based on this scope, I want to collect the variables below, with the respective tables and charts generated for the analyses, always maintaining shades of blue in all tables and charts.

To define the variables I need to address, I used the 2023 questionnaire as a base, so some of the variables from this year may be missing in the previous years, requiring adjustments in the data source."

Blocks 01 and 02 - Unit identification and characterization

  • Variable 01 - State (UF) of the units
  • Variable 02 - Municipalities of the units
  • Variable 03 - State or municipal management
  • Variable 04 - Nature of the unit - governmental or non-governmental
  • Variable 05 - Does the unit receive financial resources from Social Assistance?
  • Variable 06 - Registration/record in councils

BLOCKS 03 and 04 - Physical structure and Services and Activities

  • Variable 07 - Accessibility conditions for people with disabilities and elderly people
  • Variable 08 - Presence of public transport in the area where the unit is located
  • Variable 09 - Service capacity per shift
  • Variable 10 - How many days per week do the elderly attend the unit?
  • Variable 11 - How many hours per day do the elderly remain at the unit?
  • Variable 12 - Presence of a user waiting list

BLOCK 05 - User profile

  • Variable 13 - Number of elderly people with disabilities or dependency attended
  • Variable 14 - Number of people by type of listed disability

Development

I. Data Collection

  1. I collected the data for the 5 years to be analyzed.
  2. I created a folder structure to facilitate access to the collected data.

II. Initial Data Study

  1. Each year has its own structure for the provided data, so it was necessary to identify the applied questionnaires and the data sources of the collected responses.

    2. In 2023, 24 files were provided.

    In 2022, 22 files.

    In 2021, 21 files.

    In 2020, 23 files.

    In 2019, 21 files.

  2. Among the provided files, each year had a data source with the responses from the applied questionnaires.

    For all years, the data source was available in ".csv", ".sav", and ".xlsx" formats.

  3. Each year had a unique questionnaire, with changes in structure, question order, inclusion or removal of questions, or changes in answer options.
  4. Regarding the variables in each data source:

    In 2023, the data source had 563 variables (columns).

    In 2022 and 2021, each had 663 variables.

    In 2020, 668 variables.

    In 2019, 571 variables.

  5. For all years, a dictionary was provided for the data sources.
  6. Before starting the data transformation process for analysis, it was necessary to identify all the variables requested by the client in each data source. To support this process, a Notion documentation was created to guide the entire work.

III. Organization and Pre-transformation of Data

  1. After identifying each variable, I gathered the 5 data sources, placed them in the same directory, and renamed each one according to the respective year (2023, 2022, ...).
  2. With the directories organized, I created an Excel file responsible for transforming the data and generating a clean and organized data source.
  3. The following tools were chosen for data transformation:

    Power Query: A powerful data transformation tool.

    Python with Pandas: For final adjustments and data unification.

Data Transformation

  1. In Power Query, the following adjustments were made:

    1. Data types of the used variables were corrected;

    2. Variables were renamed to be standardized across the 5 years.

    3. Many responses were stored as numeric values, so I replaced them to facilitate identification of the selected options.

    4. Clean and organized data sources were generated for each year.

  2. After generating the new data sources, I moved the transformation process to Python due to its ease in removing unused columns and merging the data into a single file. The steps followed were:

    1. Importing the data into Jupyter Notebook.

    2. Creating a DataFrame with only the desired variables, already organized in the Notion documentation.

    3. Merging the data into a single data source.

    4. Exporting the new file to Excel.

Data Analysis

  1. Since the client requested the delivery in Excel, the entire data analysis was carried out there.
  2. In Excel, a "Summary" tab was created explaining the entire project.
  3. Each requested variable was analyzed in a separate Excel tab.
  4. The clean and organized data source was delivered along with the analysis.
  5. For each variable, quantitative, percentage, and year-over-year variation tables were generated, along with charts for all tables.
  6. For quantitative tables, two bar charts with inverted axes were created, so the client could choose which one to use in their report.
  7. For percentage tables, stacked bar charts were generated and, for those with fewer variables, pie charts were also created.
  8. For all variables, percentage variation tables between years were created.
  9. Some variables contained geographic data in the analysis, and for these, heat maps with quantities were generated.
  10. For all variables, information about the study, table guidelines, and key insights were provided.
  11. At the end of the analysis, the Excel file contained 159 tables and 214 charts.

Results

The project was a success. The client was very satisfied and provided great feedback.
